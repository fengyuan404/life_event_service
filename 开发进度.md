# 人生大事服务管理系统开发进度

## 1. 项目定位

本项目是数据库课程设计配套的全栈演示系统，围绕现有数据库 `life_event_service` 实现人生大事服务管理后台。

开发目标优先级：

1. 能启动。
2. 能演示。
3. 能完成核心业务闭环。
4. 严格贴合 `前后端项目开发大纲.md`。
5. 不改变 `人生大事服务管理系统数据库脚本.sql` 中的数据库表结构。

当前已新建两个工程：

- `backend/`：Spring Boot 后端，端口 `8080`。
- `frontend/`：Vue 前端，端口 `5173`。

## 4. 数据库规范

必须继续沿用现有 SQL 脚本：

- `人生大事服务管理系统数据库脚本.sql`

重要约束：

- 不要随意改表结构。
- 不要删除外键历史数据。
- 删除类操作优先采用业务状态更新或受限删除。
- 租赁创建成功后必须同步更新墓穴状态。

核心表：

- `staff`：员工账号。
- `family`：家属信息。
- `deceased`：逝者档案。
- `grave_area`：墓区。
- `grave`：墓穴。
- `rent_record`：租赁记录。
- `payment`：缴费记录。
- `sacrifice_book`：祭扫预约。

数据库初始化：

```bash
mysql -u root -p < 人生大事服务管理系统数据库脚本.sql
```

默认后端连接配置在：

- `backend/src/main/resources/application.yml`

默认配置：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/life_event_service?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&useSSL=false
    username: ${DB_USERNAME:root}
    password: ${DB_PASSWORD:123456}
```

如果本机 MySQL 密码不是 `123456`，修改 `application.yml` 或设置环境变量：

- `DB_USERNAME`
- `DB_PASSWORD`

## 10. 核心业务规则

租赁办理：

- 必须校验家属存在。
- 必须校验逝者存在。
- 必须校验墓穴存在。
- 必须校验经办员工存在。
- 墓穴状态必须为 `empty`。
- 到期日期必须晚于开始日期。
- 创建租赁成功后，墓穴状态更新为 `rented`。

缴费登记：

- 必须关联已存在租赁订单。
- 必须校验收款员工存在。
- 缴费金额必须大于 `0`。
- 票据编号 `invoice_no` 由数据库唯一约束保证不重复。

祭扫预约：

- 必须关联家属、墓穴、员工。
- 预约人数必须大于 `0`。
- 新建预约默认状态为 `booked`。
- 只有 `booked` 状态可以核销。
- 已核销预约不能取消。

家属删除：

- 如果家属已有逝者档案，不允许物理删除。
- 这是为了避免破坏外键历史数据。

## 14. 项目规范

后续开发请遵守：

- 不改变数据库表结构，除非用户明确要求。
- 不引入 Spring Security，保持课程设计演示复杂度可控。
- 不引入复杂 JWT 刷新机制，当前 token 内存会话即可。
- API 继续使用统一响应格式。
- 新增列表接口应支持 `keyword`、`page`、`size`。
- 前端请求统一走 `src/api/modules.js`。
- 前端错误提示统一依赖 `src/api/http.js`。
- 表单统一使用 Element Plus Dialog + Form。
- 金额、人数、日期等字段必须做基础校验。
- 删除类操作必须二次确认。
- 涉及历史数据的删除优先用状态更新或业务限制。
- UI 继续保持后台管理风格，清晰、紧凑、可演示。

## 16. 当前已知注意点

1. 后端 token 存在内存里，重启后需要重新登录。
2. 密码没有加密，只是和 `staff.password_hash` 字段直接比较，符合课程设计演示目标。
3. 前端构建产物较大，Vite 会提示 chunk 超过 500 kB，原因是 Element Plus 和 ECharts 体积较大，不影响演示。
4. 当前没有单元测试，已做构建和启动验证。
5. 后端需要 MySQL 已启动，且数据库脚本已执行。
6. 当前 `application.yml` 默认 MySQL 密码是 `123456`。
7. SQL 脚本后半部分包含查询、更新、删除、授权示例，初始化数据库时应确认本机 MySQL 用户权限允许执行。

1. 执行 SQL 脚本初始化数据库。
